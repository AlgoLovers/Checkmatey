# 제품 비전 — Checkmatey

## 한 줄

체스를 **처음 배우는 사람**이 규칙부터 실전까지 단계적으로 익히는 안드로이드 학습 앱. 폰·태블릿 모두.

## 원칙

- **학습 우선**: 그냥 두는 앱이 아니라 *왜 그 수가 좋은지/나쁜지*를 가르친다.
- **작고 결정적인 코어**: 체스 규칙은 순수 Kotlin + 테스트로 못박는다(모델 추측 금지).
- **한 코드, 두 폼팩터**: 적응형 레이아웃으로 폰·태블릿을 동시에.
- **점진적 자율 개발**: 루프 엔지니어링으로 만들되 L1(보고)에서 시작해 신뢰가 쌓이면 올린다.

## 탭 (초기 IA)

- **Learn** — 규칙 → 기물 이동 → 기본 전술 → 오프닝 레슨.
- **Play** — 보드에서 두기. (초기: 로컬 2인/자기연습, 이후: 난이도별 봇)
- **Puzzles** — 전술 퍼즐(레이팅), 데일리 퍼즐, 진행 저장.
- **Profile** — 진행·레이팅·설정(다크모드/보드 테마/기물 세트).

## 기능 로드맵 (권위 있는 마일스톤 목록은 STATE.md)

- **M1 체스 규칙 엔진**: 합법수·체크·체크메이트·스테일메이트·특수수(캐슬링/앙파상/승격), 전체 FEN.
- **M2 멀티엔진 + 레벨 봇**: `Engine`/`BotLevel`/`Annotator` 인터페이스 + 자체 `KotlinMinimaxEngine`.
  **≤1200을 4레벨**(탐색깊이+블런더율). 인터페이스 뒤에 나중에 Stockfish/Maia 교체 가능.
- **M3 온디바이스 힌트/해설**: 최선수 + "왜"(엔진 평가 델타 + 전술 모티프 → 템플릿 설명). 오프라인.
- **M4 기보 공부 모드(우선)**: 명국 리플레이 + 다음 수 맞히기 + 복기 + 간격 반복.
- **M5 퍼즐** · **M6 오프닝 훈련** · **M7 Profile/영속화**.

## 엔진·데이터 방침

- **엔진**: 퍼미시브 우선. 자체 Kotlin 미니맥스로 시작(~1200 충분). 강한 분석이 필요해지면
  `Engine` 인터페이스 뒤에 Stockfish(GPL) 등을 옵션으로. 앱 코드는 안 바뀐다.
- **레벨(≤1200)**: 새싹~400 / 초급~700 / 중급~1000 / 도전~1200. (탐색깊이 + 블런더 확률)
- **힌트 설명**: 온디바이스 규칙/템플릿(결정적, 무료, 오프라인). 유려한 문장 코칭은 나중 LLM 옵션.
- **데이터**: **Lichess Open Database (CC0, 퍼블릭 도메인)** — 게임/퍼즐/오프닝. 상용에도 사용 가능.
  역사적 명국은 수순(사실)만 사용하고 주석은 자체 생성한다(남의 주석/해설은 저작권).

## 나중(백로그)

- 강한 분석 엔진(Stockfish) 옵션 통합: 평가바, 정밀 블런더 감지.
- 사람같은 상대(Maia) 옵션.
- 온라인 대전 / 계정 동기화, 접근성(좌표 읽기), 다국어(ko/en).

## 성공 신호 (측정 대상)

- 첫 레슨 완주율, 퍼즐 정답률 추이(레이팅 상승), 재방문(리텐션).
- 각 마일스톤은 "관측 가능한 결과 하나"로 정의한다(CLAUDE.md 규칙 3).
