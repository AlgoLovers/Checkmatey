## 제품 목표

체스 **실력 향상**이 목적. 두 축: (1) 컴퓨터와 실전(레벨별), (2) **명국 기보 공부**(바둑식 —
따라두기·복기·반복). 온디바이스 힌트로 최선수와 이유를 설명. 폰+태블릿.

## 결정 로그 (Decision Log)

- **2026-07-17** · 스택 = Kotlin + Jetpack Compose (Material 3), minSdk 24 / target 35.
- **2026-07-17** · 폰+태블릿 = `NavigationSuiteScaffold`(자동 적응) + `WindowSizeClass`.
- **2026-07-17** · 원격 = github.com/AlgoLovers/Checkmatey (public). CI/lint 초록불 유지.
- **2026-07-17** · 개발 방식 = Loop Engineering. 개발 루프는 L2(브랜치→PR, 머지는 사람).
- **2026-07-17** · **라이선스 = 퍼미시브 우선.** 자체 Kotlin 엔진을 기본 번들. Stockfish/Maia(GPL)는
  `Engine` 인터페이스 뒤 옵션으로만(나중). ~1200까지는 자체 엔진으로 충분. (사용자 결정)
- **2026-07-17** · **컴퓨터 상대 = ≤1200, 4레벨**(탐색깊이+블런더율로 구현). (사용자 결정)
- **2026-07-17** · **기보 공부 모드 우선순위 = 명국 리플레이 + 다음수 맞히기.** 그다음 퍼즐→오프닝. (사용자 결정)
- **2026-07-17** · **힌트 '왜' 설명 = 온디바이스 규칙/템플릿**(엔진평가+모티프). LLM은 나중 옵션. (사용자 결정)
- **2026-07-17** · **학습 데이터 = Lichess Open Database (CC0, 퍼블릭 도메인)** — 게임/퍼즐/오프닝.
  명국은 수순만 사용(주석은 자체 생성). 상용에도 사용 가능.
- **2026-07-17** · **핵심 설계 원칙 = "모든 것을 레이팅 기반으로"**(사용자 강조). 상대 레벨은 항상 Elo 명시,
  향후 **사용자 본인 Elo 추적**(대국·퍼즐 결과로 갱신) + 레이팅 기반 퍼즐/매칭. → Profile(M7)에서 사용자 레이팅 도입.
